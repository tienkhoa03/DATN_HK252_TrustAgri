# Frontend UI — Con trỏ tới kế hoạch triển khai

Nội dung kế hoạch triển khai chi tiết theo định dạng **Kiro** đã được tách và chuẩn hóa trong cùng thư mục:

| Tài liệu | Mục đích |
|----------|----------|
| [requirements.md](./requirements.md) | Yêu cầu có **tiêu chí chấp nhận** (routing sản phẩm, tầng API, đồng bộ với spec refactor thư mục màn hình + design system Zalo UI). |
| [design.md](./design.md) | Kiến trúc, **bảng route**, HTTP/WebSocket, ánh xạ màn hình → service, state. |
| [tasks.md](./tasks.md) | **Kế hoạch triển khai** dạng checklist — thay thế bản nháp dài của file này. |

**Tham chiếu ngoài**

- API backend: `specs/backend-api-specification/design.md`
- Tiến độ backend (tham khảo): `specs/backend-api-specification/tasks.md`
- Spec đã hoàn thành trong repo FE: `fe/specs/screen-folder-refactoring`, `fe/specs/zalo-ui-design-system`

Phiên bản trước của tài liệu này đã được **gộp và tái cấu trúc** vào `tasks.md` cùng `requirements.md` / `design.md`.
